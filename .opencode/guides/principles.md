# Global Principles

Cross-cutting principles for all work in this project.

## Software design

- **SOLID** — Single responsibility, Open-closed, Liskov substitution,
  Interface segregation, Dependency inversion
- **DRY** — Don't Repeat Yourself; extract reusable code
- **KISS** — Keep It Simple; prefer readability over cleverness
- **YAGNI** — You Ain't Gonna Need It; don't add functionality until necessary

## Git & commits

Follow **Conventional Commits** (<https://www.conventionalcommits.org/>):

```text
<type>(<scope>): <description>

<body>
```

Allowed types: `feat`, `fix`, `chore`, `refactor`, `docs`, `test`,
`style`, `perf`, `ci`, `build`.
